# plans/

本目录用于保存玉子手机扩展尚未落地或仍在演进中的规划文档。

## 边界

- 稳定事实、当前架构、已落地维护规则写入 [`docs/`](../docs) 或 [`docs/reference/`](../docs/reference)。
- 构建、检查、发布和 `dist/` 提交规则以 [`BUILD.md`](../BUILD.md) 为准。
- 本目录只放计划、方案、迁移路线和未完成事项；计划落地后，必须把会影响后续维护的稳定结论同步回 [`docs/architecture-guide.md`](../docs/architecture-guide.md) 或对应参考文档。

## 当前状态

当前没有需要单独展开的计划文档。保留本目录是为了让文档中的 [`plans/`](.) 链接指向真实位置，并明确未实施计划不得混入稳定架构说明。
