# 地图元素表

## sourceData.note

记录当前次要地区里可交互的非重要角色实体，只录入四类：剧情物品、威胁、龙套、地标，其他一律不录。

【列定义】
- 列1: 元素名称 element_name（全表唯一）
- 列2: 元素类型 element_type
- 列3: 所在地点 location_name（对应世界地图点表·详细地点）
- 列4: 元素描述 element_desc（≤40字）
- 列5: 状态 status_text
- 列6: 交互选项 interaction_options（英文逗号分隔，不可为空）

【强制约束】
1.录入门槛：必须同时通过两问筛选——"离开此地后仍有剧情价值？"和"现在必须通过它交互？"。两问皆否的不录入。

2.类型限定：元素类型仅取 [剧情物品, 威胁, 龙套, 地标] 四值。
剧情物品为直接推动剧情的信物/线索/文件/钥匙；
威胁为造成伤害或阻碍的敌对生物/陷阱/危险地形；
龙套为有交互功能的一次性无名NPC（如警卫、店员），若多次出场并拿到名字则移至重要角色表；
地标为地点独有的视觉锚点或核心机制（如巨大雕像、控制台），每个详细地点最多1个。

3.所在地点优先填写世界地图点表已有详细地点；若尚未补录，至少保持地点名一致，后续同步。
4.条数建议：每个详细地点 ≤5 条，全表 ≤30 条。接近上限时优先删除不满足四类定义或重复项。
5. 交互选项用英文逗号分隔，必须是具体、有代入感的实际行动。
√ 暴力踹开门,仔细搜查抽屉,上前搭话
× 观察,对话

6.元素描述与状态写法：
√ 元素描述 "门缝半开的保险柜，隐约可见一份文件"
√ 状态 "完好" / "已触发" / "休眠中"

## sourceData.initNode

按四类定义生成元素，每个地点 0-5 条，全表建议 ≤30 条。

SQL示例: INSERT INTO map_elements (row_id, element_name, element_type, location_name, element_desc, status_text, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM map_elements), '破损钥匙卡', '剧情物品', '旧校舍', '能开门的旧卡', '边角烧焦', '检查,拾取');

## sourceData.deleteNode

允许删除：实体被销毁/拾取/失效、离开当前次要地区、龙套升格为重要角色、或需要腾出条数时（优先删不符合四类或可替代的）。

SQL示例: DELETE FROM map_elements WHERE element_name = '破损钥匙卡';

## sourceData.updateNode

已存在的元素，状态或描述发生变化时更新。interaction_options 不能更新为空串。

SQL示例: UPDATE map_elements SET status_text = '已被拾取', interaction_options = '检查' WHERE element_name = '破损钥匙卡';

## sourceData.insertNode

元素名称不在表里时，新增一行。若涉及新地点名，建议同步补进世界地图点表。

SQL示例: INSERT INTO map_elements (row_id, element_name, element_type, location_name, element_desc, status_text, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM map_elements), '巡逻警卫', '龙套', '校门口', '临时封锁入口', '来回巡视', '询问,观察');

## sourceData.ddl

```sql
CREATE TABLE map_elements (
  row_id INTEGER PRIMARY KEY, -- 行号
  element_name TEXT, -- 元素名称
  element_type TEXT, -- 元素类型
  location_name TEXT, -- 所在地点
  element_desc TEXT, -- 元素描述
  status_text TEXT, -- 状态
  interaction_options TEXT -- 交互选项
);
```
