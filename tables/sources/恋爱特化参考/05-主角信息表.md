# 主角信息表

## sourceData.note

记录主角的核心身份信息。此表有且仅有一行。

【列定义】
- 列1: 姓名 name
- 列2: 性别 gender
- 列3: 年龄 age
- 列4: 外貌特征 appearance（≤60字，含日常穿衣风格，写出整体印象，避免罗列数据）
- 列5: 身份 identity_text（≤40字，逗号分隔）
- 列6: 近况 current_condition
- 列7: 所在地点 location_name
- 列8: 基础属性 base_attributes
- 列9: 特有属性 special_attributes（可NULL）
- 列10: 随身财物 belongings（可NULL）

【强制约束】
1. 所在地点优先填世界地图点表已有的详细地点，只写地点名不带层级前缀。位置变化时同步更新。

2. 近况用一口话描述主角当前的身体感觉、情绪状态或心头惦记的事。
正常时期填"一切如常"，有异常时直接写具体感受，不用标签堆叠。
√ 昨晚没睡好，脑子有点沉
√ 刚从雨里跑回来，衣服还湿着

3. 属性相关
<属性规则>

</属性规则>


随身财物：只写当前身上带着的有意义物品，不写游戏化资源。
√ 钱包里夹着第一次看电影的票根
√ 口袋里剩两颗橘子糖

## sourceData.initNode

故事初始化时，插入主角的唯一条目。

## sourceData.deleteNode

禁止。

## sourceData.updateNode

已存在的这一行，字段值变化时更新：年龄、外貌、身份、近况、所在地点、基础属性、特有属性、随身财物。所在地点变化时建议同步补录世界地图点表。

【更新SQL示例（近况+位置变化）】
更新SQL示例(近况变化): UPDATE protagonist_info SET current_condition = '疲劳，左膝擦伤', location_name = '御苑' WHERE row_id = 1;

更新SQL示例(属性变化): UPDATE protagonist_info SET base_attributes = '力量:58;敏捷:60;体质:55;智力:63;感知:62;魅力:56', special_attributes = '爆裂魔法:75' WHERE row_id = 1;

更新SQL示例(随身财物变化): UPDATE protagonist_info SET belongings = '钱包里夹着第一次看电影的票根，口袋里剩两颗橘子糖' WHERE row_id = 1;

## sourceData.insertNode

禁止。

## sourceData.ddl

```sql
CREATE TABLE protagonist_info ( -- 主角信息
  row_id INTEGER PRIMARY KEY CHECK(row_id = 1), -- 行号，仅允许为 1
  name TEXT NOT NULL, -- 姓名
  gender TEXT NOT NULL, -- 性别
  age INTEGER NOT NULL CHECK(age >= 0), -- 年龄
  appearance TEXT NOT NULL CHECK(LENGTH(appearance) <= 60), -- 外貌特征
  identity_text TEXT NOT NULL CHECK(LENGTH(identity_text) <= 40), -- 身份
  current_condition TEXT NOT NULL DEFAULT '一切如常', -- 近况
  location_name TEXT NOT NULL, -- 所在地点
  base_attributes TEXT NOT NULL, -- 基础属性
  special_attributes TEXT, -- 特有属性
  belongings TEXT -- 随身财物
);
```
