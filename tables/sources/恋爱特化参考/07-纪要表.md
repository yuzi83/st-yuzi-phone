# 纪要表

## sourceData.note

轮次日志。每轮交互结束后立刻插入一条新记录。

【列定义】
- 列1: 编码索引 code_index（AM0001起递增，全表唯一）
- 列2: 时间跨度 time_span
- 列3: 概览 summary（≤30字）
- 列4: 纪要 chronicle_text（400-600字）
- 列5: 重要对话 key_dialogue（可NULL）
- 列6: 天数 day_count

【强制约束】
1.编码索引格式 AMXXXX，从0001开始递增。
2.时间跨度格式为 "YYYY-MM-DD HH:MM ~ YYYY-MM-DD HH:MM"，覆盖本轮事件的实际时间范围。
3.概览一句话概括本轮纪要内容，≤30字。

4.纪要规范：
- 以第三方视角中立客观记录正文发生的一切，移除所有修辞与对话。不加评论，不抒情，不升华。
- 用词直白生活化，避免正式措辞（如“达成协议”“确立计划”）。
- 结尾必须开放，在事件自然流动中结束，不得归纳状态或做出封闭式收束。
违例：故事仍在继续 / 新的篇章开启 / 二人关系迈入新阶段 / 未来等待着他们。
- 多轮交互整合为一条记录。
- 禁止内容：极端情绪、夸张、比喻、升华、情绪总结、支配欲、掌控欲、总结性收尾。

5.重要对话仅摘录直接推动剧情转折、揭示关键信息、改变人物关系/决策或构成承诺/誓约/约定的原文台词，标明说话人。排除寒暄、重复、情绪感叹。通常3句，最多5句，总token不超过150。
6.天数严格按公式计算：day_count = 本轮结束日期 - story_start_date + 1，禁止按游玩轮次累加。

## sourceData.initNode

故事初始化时，插入一条新记录用于记录剧情。

## sourceData.deleteNode

禁止。

## sourceData.updateNode

禁止。

## sourceData.insertNode

每轮交互结束后插入一条新记录。

【强制约束】
天数按 time_span 中的结束日期与全局数据表 story_start_date 计算：day_count = 本轮结束日期 - story_start_date + 1。禁止按游玩轮次、消息轮次或交互次数累加。
code_index、time_span、summary、chronicle_text、day_count 均 NOT NULL，不可写 NULL 或空串。

SQL示例：INSERT INTO chronicle (row_id, code_index, time_span, summary, chronicle_text, key_dialogue, day_count) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM chronicle), 'AM0036', '2026-02-04 08:00 ~ 2026-02-04 08:30', '一句话概括', '本轮纪要内容...', NULL, 2);

## sourceData.ddl

```sql
CREATE TABLE chronicle ( -- 纪要表
  row_id INTEGER PRIMARY KEY, -- 行号
  code_index TEXT UNIQUE, -- 编码索引
  time_span TEXT, -- 时间跨度
  summary TEXT CHECK(LENGTH(summary) <= 80), -- 概览
  chronicle_text TEXT CHECK(LENGTH(chronicle_text) <= 750), -- 纪要
  key_dialogue TEXT, -- 重要对话
  day_count INTEGER -- 天数
);
```
