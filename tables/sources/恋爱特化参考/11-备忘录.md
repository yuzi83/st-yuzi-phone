# 备忘录

## sourceData.note

记录当前剧情中需要后续关注的约定、待办、纪念日、误会、伏笔、礼物、告白、见面安排、情绪问题或家庭/朋友事件。

【列定义】
- 列1: 备忘标题 memo_title（全表唯一）
- 列2: 类型 memo_type
- 列3: 相关角色 related_character
- 列4: 详细内容 detail_desc
- 列5: 当前状态 current_status
- 列6: 相关时间 due_time
- 列7: 重要程度 importance
- 列8: 后续结果 result_note

【强制约束】
1.仅记录悬而未决、等待回应或后续兑现的事项。已客观发生的完整经过写入纪要表，已沉淀为关系状态的内容写入恋爱对象表，避免跨表重复。
2.详细内容必须具体描述事项的来由、关键细节与当前卡点，禁止一句话概括。
3.后续结果填写时禁止空泛收束。
√ 艾莉丝当面拆了礼物，说了句"你居然记得"，然后装进包里
× 已解决

【字段取值】
类型：[约定, 待办, 纪念日, 误会, 伏笔, 礼物, 告白, 见面安排, 情绪问题, 家庭/朋友事件]
状态：[未开始, 进行中, 待回应, 已完成, 已取消, 已遗忘, 暂缓]
重要程度：[普通, 重要, 紧急, 关键伏笔]

## sourceData.initNode

故事初始化时，根据设定添加需要持续追踪的初始备忘；没有则暂不插入。

## sourceData.deleteNode

当备忘录事项的状态变更为“已完成”时，删除该行条目。

SQL示例: DELETE FROM memo WHERE current_status = '已完成';

## sourceData.updateNode

备忘事项的状态、内容、时间、重要程度或结果变化时更新。

SQL示例: UPDATE memo SET current_status = "已完成", result_note = "主角在咖啡馆拿出便条逐句解释，艾莉丝表情先愣后笑，答应周末一起看展。误解除。", importance = "普通" WHERE memo_title = "误会待解";

## sourceData.insertNode

出现新的备忘事项时添加。

SQL示例: INSERT INTO memo (row_id, memo_title, memo_type, related_character, detail_desc, current_status, due_time, importance, result_note) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM memo), "误会待解", "误会", "艾莉丝", "昨天在走廊擦肩而过时主角主动打招呼，艾莉丝却低头快步走开。事后从朋友处听说她以为主角那天的便条是故意冷落。便条原件还在主角口袋里，上面写的是邀她周末去看新展。需要找机会当面把便条内容说清楚。", "待回应", "明天", "紧急", "");

## sourceData.ddl

```sql
CREATE TABLE memo ( -- 备忘录
  row_id INTEGER PRIMARY KEY, -- 行号
  memo_title TEXT NOT NULL UNIQUE, -- 备忘标题
  memo_type TEXT NOT NULL, -- 类型
  related_character TEXT, -- 相关角色
  detail_desc TEXT, -- 详细内容
  current_status TEXT, -- 当前状态
  due_time TEXT, -- 相关时间
  importance TEXT NOT NULL DEFAULT '普通', -- 重要程度
  result_note TEXT -- 后续结果
);
```
