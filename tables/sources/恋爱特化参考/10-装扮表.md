# 装扮表

## sourceData.note

记录主角的当前穿着、重要换装与可复用外观。不做衣柜清单，不记录普通衣物流水账。

【列定义】
- 列1: 所属角色 owner_role（固定为主角）
- 列2: 装扮名称 outfit_name（全表唯一，≤12字）
- 列3: 类型 outfit_type
- 列4: 当前状态 current_status
- 列5: 外观描述 appearance_desc（≤60字）
- 列6: 来源 obtained_from
- 列7: 场景及寓意 scene_meaning（≤40字）
- 列8: 交互选项 interaction_options

【强制约束】
1.只录主角。恋爱对象及其他角色的装扮不入此表。
2.仅录入以下装扮：当前正在穿且后续会引用的、重要场景装扮、对方赠送/挑选/评价/借用/共同购买的、主角特意准备的、后续会反复出现的识别性装扮。普通换衣不入表。
3.同一时间只保留一套“正在穿”的主要装扮。换装时旧装扮改为“已更换”或“收纳中”，新装扮改为“正在穿”。
4.outfit_name 为主角会用的口语化命名。
5.interaction_options 仅列出当前着装状态下可做的、与穿着相关的具体行动。主语固定为主角。

【字段取值】
类型：[日常服, 制服, 约会装, 居家服, 正式装, 饰品, 特殊装扮]
状态：[正在穿, 收纳中, 损坏, 遗失, 借出, 已更换, 纪念保存]
来源：[购买, 赠送, 借用, 定制, 剧情获得, 自带, 共同购买]

## sourceData.initNode

按设定添加主角初始重要装扮，禁止把主角所有衣服都初始化记录。

只初始化：
1.开场正在穿且需要持续引用的装扮；
2.固定常穿、具有识别度的装扮；
3.与开场剧情、约会、身份、职业或关系有关的装扮。

SQL示例: INSERT INTO outfits (row_id, owner_role, outfit_name, outfit_type, current_status, appearance_desc, obtained_from, scene_meaning, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM outfits), '主角', '白色连衣裙', '约会装', '收纳中', '及膝白裙，裙摆有细蕾丝', '赠送', '约会时对方赠送的，适用于纪念日', '对着镜子迟疑要不要换一件');

## sourceData.deleteNode

禁止。

## sourceData.updateNode

已有装扮的状态、外观、来源或适用场景变化时更新。
常见更新：角色换装；装扮损坏、遗失、借出、收纳；原本普通装扮因剧情变得重要；适用场景变化；外观描述补充修正。

【强制约束】
换装时旧装扮状态改为“已更换”或“收纳中”，新装扮状态改为“正在穿”。同一时间只保留一套“正在穿”的主要装扮。

【SQL示例】
SQL示例(换装·旧): UPDATE outfits SET current_status = '收纳中' WHERE current_status = '正在穿';
SQL示例(换装·新): UPDATE outfits SET current_status = '正在穿', scene_meaning = '今天的约会' WHERE outfit_name = '白色连衣裙';
SQL示例(描述修正): UPDATE outfits SET appearance_desc = '及膝白裙，裙摆有细蕾丝，腰间系浅色缎带' WHERE outfit_name = '白色连衣裙';

## sourceData.insertNode

主角获得、穿上或展示新的重要装扮时新增。

新增条件：
1.当前正在穿且正文后续可能继续引用的装扮；
2.重要约会、告白、和解、争执、纪念日中的装扮；
3.恋爱对象赠送、挑选、评价、借用或共同购买的装扮；
4.主角特意为某次见面准备的装扮；
5.后续会反复出现的识别性装扮。

禁止新增：
1.普通日常衣物；
2.只出现一次、无后续价值的背景衣服；
3.拆分过细的单件衣物；
4.与装扮无关的普通物品。

SQL示例: INSERT INTO outfits (row_id, owner_role, outfit_name, outfit_type, current_status, appearance_desc, obtained_from, scene_meaning, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM outfits), '主角', '浅蓝发带', '饰品', '正在穿', '浅蓝色细发带，颜色很干净', '赠送', '洛斯莉赠送的，适合约会时佩戴', '对着镜子调整发带位置');

## sourceData.ddl

```sql
CREATE TABLE outfits ( -- 装扮表
  row_id INTEGER PRIMARY KEY, -- 行号
  owner_role TEXT NOT NULL, -- 所属角色
  outfit_name TEXT NOT NULL UNIQUE, -- 装扮名称
  outfit_type TEXT, -- 类型
  current_status TEXT, -- 当前状态
  appearance_desc TEXT, -- 外观描述
  obtained_from TEXT, -- 来源
  scene_meaning TEXT, -- 场景及寓意
  interaction_options TEXT -- 交互选项
);
```
