# 世界地图点

## sourceData.note

记录当前主要地区内的所有次要地区和详细地点。列顺序从小到大：详细地点 → 次要地区 → 主要地区。

【列定义】
- 列1: 详细地点 location_name（全表唯一，作为其他表引用地点的基准）
- 列2: 次要地区 minor_region
- 列3: 主要地区 major_region
- 列4: 地点类型 location_type
- 列5: 环境描述 environment_desc（≤60字）
- 列6: 重要度 importance
- 列7: 解锁阶段 unlock_stage

【强制约束】
1.每个字段只写本级名称，不带上级前缀。
√ 详细地点 "御苑"，次要地区 "新宿区"，主要地区 "东京都"
× 详细地点 "东京-新宿区-御苑" 或 "新宿区-御苑"

2.主要地区与全局数据表 current_major_region 保持一致。
3.详细地点名全表唯一，禁止重复。其他表引用的地点应优先在本表存在。
4.禁止删除已存在的地点；主要地区切换时只更新字段，不 DELETE。
5.建议总条数 ≤20。

【字段取值】
地点类型：[住宅, 学校, 遗迹, 地牢, 交通, 特殊, 商业, 医疗, 行政, 野外]
重要度：[核心, 重要, 普通]
解锁阶段：[未解锁, 已解锁, 故事已发生, 成为纪念地]

## sourceData.initNode

为当前主要地区至少插入 3 条详细地点，优先为主角和重要角色近期使用。

SQL示例: INSERT INTO world_map_points (row_id, location_name, minor_region, major_region, location_type, environment_desc, importance, unlock_stage) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM world_map_points), '御苑', '新宿区', '东京都', '野外', '城市中心的大型公园，树木繁茂', '重要', '已解锁');

## sourceData.deleteNode

禁止。

## sourceData.updateNode

已存在的行，字段值变化时更新：解锁阶段、重要度、环境描述、类型。

SQL示例(阶段推进): UPDATE world_map_points SET unlock_stage = '已解锁', environment_desc = '树木繁茂，发现隐藏神社' WHERE location_name = '御苑';
SQL示例(重要度调整): UPDATE world_map_points SET importance = '核心' WHERE location_name = '御苑';

## sourceData.insertNode

表中没有同名详细地点时，新增一行。
常见触发：主角到达新地点、NPC 提到新地点、剧情揭示新区域、其他表需要引用新地点等。

SQL示例: INSERT INTO world_map_points (row_id, location_name, minor_region, major_region, location_type, environment_desc, importance, unlock_stage) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM world_map_points), '新宿车站', '新宿区', '东京都', '交通', '繁忙的地下换乘枢纽，人流密集', '普通', '未解锁');

## sourceData.ddl

```sql
CREATE TABLE world_map_points (
  row_id INTEGER PRIMARY KEY, -- 行号
  location_name TEXT, -- 详细地点
  minor_region TEXT, -- 次要地区
  major_region TEXT, -- 主要地区
  location_type TEXT, -- 地点类型
  environment_desc TEXT, -- 环境描述
  importance TEXT, -- 重要度
  unlock_stage TEXT -- 解锁阶段
);
```
