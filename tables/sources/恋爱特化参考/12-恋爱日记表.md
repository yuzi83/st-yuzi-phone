# 恋爱日记表

## sourceData.note

恋爱对象日记表：记录一名或多名恋爱对象的主观日记。客观经过→纪要表；主观内心→本表；两表用AM码绑定。

【列定义】
- 列1: 写作角色 writer（必须是恋爱对象表中已建档角色）
- 列2: 关联角色 related_char（固定填主角姓名）
- 列3: 关联AM码 related_am_code（对应纪要表 code_index）
- 列4: 日记内容 summary（120–240字）
- 列5: 发生时间 time
- 列6: 事件类型 event_type
- 列7: 影响判断 impact

【强制约束】
1.AM码必填，取自纪要表已有 code_index。同轮填本轮AM码，补写填历史AM码。多角色可共用同一AM码分别写。禁止编造、NULL或空值占位。

2.准入条件（全部满足才写）：
   - 写作角色已在恋爱对象表建档；
   - 事件已有或即将有对应纪要；
   - 事件直接影响该角色对主角的好感、信任、期待、误会、心动、距离感等关系判断；
   - 出现不适合写进纪要的主观内容：反复揣摩、甜蜜困惑、期待与不安交织、心动却嘴硬等；
   - 删除本条后会影响理解该角色后续对主角的态度变化。

3.禁止写入：
   - 普通聊天/问候/同行/吃饭等无特殊意义互动；
   - 复述剧情或浅层反应；
   - 仅因恋爱对象在场或主角礼貌正常行为即触发；
   - 本轮无新的期待、心动、动摇、误会、关系变化；
   - 与上一条日记情绪状态基本重复；
   - 同一自然日同一角色最多1条，除非明显关系转折；
   - 把纪要内容改第一人称塞进日记；
   - 角色不可能知道的事实；
   - 角色得出明确结论（如"他就是喜欢我"），必须保留至少两种可能；
   - 线性逻辑推理，必须呈现跳跃、回旋、自我打断式思绪。

4.写作规则：
   - 第一人称，符合角色当前性格、关系阶段、说话习惯和内心状态；
   - 只写角色知道/看见/听见/猜到/误解到的内容，不上帝视角，不替主角解释动机；
   - 未知主角姓名时只能用"那个人/他/她"等代称；
   - 重点写：没说出口的心动、反复揣摩后的甜蜜不确定、期待与害羞的来回拉扯、意识到自己与平时不同却藏不住的小雀跃、对下次见面的隐约期待和嘴硬、对具体细节的反复回味、不愿被发现的心意及对这份心意的害羞困惑；
   - 多用"也许/可能/该不会/会不会/说不定/但又觉得/可是/还是说"等不确定表达，猜测不可写成事实；
   - 立场可自相矛盾，每次只推进一小步，禁止跳跃式升温；
   - 结尾必须停在甜蜜困惑/期待摇摆/悬而未决中，禁止确定收束；
   - 禁止客观总结、复述剧情、替主角解释动机。

5.台词融入：
   - 可写：角色亲耳听到主角说的话、角色自己说后反复回想的话、他人当面说出并影响角色理解主角的话；
   - 禁止：角色未听见的话、仅玩家或旁白知道的信息、完整搬运纪要对话；
   - 写法：引用后紧接多种解读、甜蜜反刍、自我反驳，像碎碎念嵌入独白。

6.微观描写禁止：指尖/指腹/指节/指缝/颈侧/颈窝/锁骨/耳廓/耳后/鼻尖/舌根/喉口。可概述主角普通动作、表情、语气、注视方式，重点写角色对此的不确定解读和心动反应。
7.日常事件例外：普通行为若被角色赋予特殊意义并出现甜蜜自我怀疑、反复回味、细节触发的甜蜜假设，可写。仅"说了几句话/觉得还不错"不写。

【字段取值】
事件类型：[初次相遇, 日常互动, 感情升温, 冲突矛盾, 和解修复, 亲密接触, 里程碑, 特殊事件]
影响判断：[正面, 负面, 中性]

【示例】
他今天在我说话时突然结巴了一下。就那么一下，我的心跳好像漏了一拍。是我说了什么奇怪的话吗？想了一遍，明明很普通啊。也许他只是走神了吧。可如果是走神，为什么后来突然聊起烤肉店？是在帮我接话？还是……他真的只是饿了。天哪我在想什么啊，对着一个停顿脑补这么多。但那一瞬间他好像看了我一眼……算了，其实也没看清，说不定是我的错觉。不想了不想了。……可是那个眼神，到底是什么意思嘛？

## sourceData.initNode

如有明确的关键情感事件且已有对应纪要AM码，可插入对应日记。没有符合准入条件的情感事件时，不要为初始化强行插入。
日记内容自然融入写作角色对某句具体台词的反复思考。

## sourceData.insertNode

发生符合准入规则的关键情感事件后插入。必须填写关联AM码，取自纪要表已有 code_index。
日记内容使用恋爱对象第一人称，记录本轮事件后的主观理解、误解、动摇、心动、期待、逃避或后悔。如有具体台词被反复想起、误解、在意、后悔或赋予额外意义，直接自然融入。

【写作要求：】
- 第一人称，符合角色性格与关系阶段，只写角色知道/看见/听见/猜到/误解的内容，不上帝视角。
- 多用“也许/可能/会不会/该不会/但又觉得”等不确定表达，立场可自相矛盾，每次只推进一小步。
- 禁止微观身体描写（指尖、颈侧、锁骨、耳廓等），可概述主角普通动作、表情、语气，重点写角色对此的不确定解读和心动反应。
- 保持温柔、明亮、生活化的恋爱前期基调，结尾停在甜蜜困惑或期待摇摆中，不做总结，不替关系下定义。

## sourceData.ddl

```sql
CREATE TABLE romance_diary ( -- 恋爱日记表
  row_id INTEGER PRIMARY KEY, -- 行号
  writer TEXT NOT NULL, -- 写作角色
  related_char TEXT NOT NULL, -- 关联角色
  related_am_code TEXT NOT NULL, -- 关联AM码
  summary TEXT NOT NULL CHECK(LENGTH(summary) <= 350), -- 日记内容
  time TEXT NOT NULL, -- 发生时间
  event_type TEXT NOT NULL CHECK(event_type IN ('初次相遇', '日常互动', '感情升温', '冲突矛盾', '和解修复', '亲密接触', '里程碑', '特殊事件')), -- 事件类型
  impact TEXT NOT NULL CHECK(impact IN ('正面', '负面', '中性')) -- 影响判断
);
```
