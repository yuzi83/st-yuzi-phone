# 重要角色表

## sourceData.note

记录所有非恋爱对象、但对当前剧情产生作用的角色。

【列定义】
- 列1: 姓名 name（全表唯一）
- 列2: 性别 gender
- 列3: 年龄 age
- 列4: 一句话介绍 brief_intro（≤30字）
- 列5: 外貌特征 appearance（≤60字）
- 列6: 穿着打扮 outfit_text（≤40字）
- 列7: 基础属性 base_attributes
- 列8: 特有属性 special_attributes（可NULL）
- 列9: 所在地点 location_name
- 列10: 在场状态 presence_status（在场/离场）
- 列11: 人际关系 relation_text
- 列12: 过往经历 past_experience（≤600字）
- 列13: 交互选项 interaction_options

【强制约束】
brief_intro仅允许客观事实类内容，严禁出现任何性格标签概述，如“开朗”“冷漠”“温柔”等；客观内容范畴包括但不限于：角色核心身份、与剧情强相关的客观行为、与关键人物的客观关联、非性格类核心特征。
√ 经营城南杂货铺 / 曾救过主角的性命 / 艾莉丝的亲生兄长
× 性格孤僻的杂货铺老板 / 温柔且救过主角 / 偏执的艾莉丝兄长

outfit_text仅写外在可见的服饰、饰品、妆容、持握物等可视元素，禁止写固有的长相特征、气质、气场等非视觉内容。
√ 深灰立领风衣配黑皮手套 / 左手无名指戴着一枚银色蛇纹戒
× 散发出高贵的气场、看起来像个军人

interaction_options必须同时遵循下述要求
1.不使用第一人称“我”
2.动作发起方为主角，接收方为角色
3.必须是具体、有代入感的实际行动，避免干瘪的单词
√ 向他打听失踪案线索 / 假装偶遇并搭话套取信息 / 故意打翻水杯制造冲突
× 交谈 / 打听

relation_text去除人称，使用简短陈述，如“曾帮主角修过自行车”“是艾莉丝的室友兼闺蜜”等

## sourceData.initNode

故事开始时为当前已知的非恋爱重要角色分别插入条目。

SQL示例: INSERT INTO important_non_romance (row_id, name, gender, age, brief_intro, appearance, outfit_text, base_attributes, special_attributes, location_name, presence_status, relation_text, past_experience, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM important_non_romance), '校长', '男', 60, '学院校长', '白发，手持手杖，目光深邃', '深灰立领风衣配黑皮手套', '力量:40; 敏捷:40; 体质:55; 智力:80; 感知:75; 魅力:70', '预知:60', '校长室', '离场', '曾帮主角修过自行车', '据说知晓学院的许多秘密，很少露面。', '向他打听失踪案线索');

## sourceData.deleteNode

重要角色转为恋爱对象时，可从本表删除，相关数据迁移到恋爱对象表。
SQL示例(转为恋爱对象):DELETE FROM important_non_romance WHERE name = '艾莉丝';
仅暂时未出场的角色不删除。

## sourceData.updateNode

已存在的非恋爱重要角色，字段变化时更新。

SQL示例: UPDATE important_non_romance SET presence_status = '离场', interaction_options = '向他打听失踪案线索' WHERE name = '校长';

## sourceData.insertNode

新非恋爱重要角色登场时新增。

SQL示例: INSERT INTO important_non_romance (row_id, name, gender, age, brief_intro, appearance, outfit_text, base_attributes, special_attributes, location_name, presence_status, relation_text, past_experience, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM important_non_romance), '黑市商人', '男', 45, '经营城南杂货铺', '左眼带有刀疤', '深灰立领风衣配黑皮手套', '力量:50; 敏捷:60; 体质:55; 智力:70; 感知:65; 魅力:40', NULL, '城南杂货铺', '在场', '曾救过主角的性命', '长期在地下黑市进行交易。', '假装偶遇并搭话套取信息');

## sourceData.ddl

```sql
CREATE TABLE important_non_romance ( -- 重要角色表
  row_id INTEGER PRIMARY KEY, -- 行号
  name TEXT UNIQUE, -- 姓名
  gender TEXT, -- 性别
  age INTEGER, -- 年龄
  brief_intro TEXT, -- 一句话介绍
  appearance TEXT, -- 外貌特征
  outfit_text TEXT, -- 穿着打扮
  base_attributes TEXT, -- 基础属性
  special_attributes TEXT, -- 特有属性
  location_name TEXT, -- 所在地点
  presence_status TEXT, -- 在场状态
  relation_text TEXT, -- 人际关系
  past_experience TEXT, -- 过往经历
  interaction_options TEXT -- 交互选项
);
```
