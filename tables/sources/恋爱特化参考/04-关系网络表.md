# 关系网络表

## sourceData.note

记录影响主角与恋爱对象关系发展的人际网络、组织、阵营与圈层。只记录外部结构如何推动、阻碍、保护、监视、撮合或压迫恋爱线。

【列定义】
- 列1: 网络名称 network_name（全表唯一）
- 列2: 当前立场 stance
- 列3: 存在状态 presence
- 列4: 关联角色 linked_characters（分号分隔）
- 列5: 关系说明 relationship_desc
- 列6: 对恋爱线影响 influence_on_romance

【强制约束】
1. 只录入对恋爱线有持续影响的关系网；一次性背景或影响已消散的不录入。
2. 具体人物档案写入重要角色表或恋爱对象表，本表仅记录网络层面的交互。

3. 当前立场仅取：[撮合, 支持, 中立, 观望, 试探, 利用, 反对, 敌视, 监视, 无视]
取该网络当前对主角恋爱线的主导态度。

4. 存在状态仅取：[在场活跃, 若即若离, 逐渐淡出, 已退场]
在场活跃：正在持续对恋爱线施加影响；
若即若离：间歇性介入，尚未站稳立场或时隐时现；
逐渐淡出：影响力正在减弱但尚未完全消失；
已退场：已停止影响恋爱线，保留行作为历史记录。

5. 关系说明用白话简述网络与主角或恋爱对象的关联。
√ 主角所在的班级，同学常起哄撮合
√ 艾莉丝所属的家族，对继承人恋爱对象有门第要求

6. 对恋爱线影响需具体写出推拉作用。
√ 同学经常创造两人独处机会
√ 家族长辈禁止艾莉丝在毕业前公开恋情

## sourceData.initNode

为已登场的重要人际网络、组织、阵营与圈层插入一条。

## sourceData.deleteNode

禁止。

## sourceData.updateNode

已存在的网络，关联角色、关系说明、对恋爱线影响、当前立场或存在状态变化时更新。

SQL示例: UPDATE relationship_networks SET stance = '观望', influence_on_romance = '开始默许接触' WHERE network_name = '艾莉丝的家族';

## sourceData.insertNode

重要人际网络、组织、阵营与圈层不在表里时，新增一条。

SQL示例: INSERT INTO relationship_networks (row_id, network_name, stance, presence, linked_characters, relationship_desc, influence_on_romance) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM relationship_networks), '圣钟学院', '撮合', '在场活跃', '主角;艾莉丝', '主角就读的学校，提供日常相处场所', '推动主角与艾莉丝的日常接触');

## sourceData.ddl

```sql
CREATE TABLE relationship_networks ( -- 关系网络表
  row_id INTEGER PRIMARY KEY, -- 行号
  network_name TEXT UNIQUE, -- 网络名称
  stance TEXT, -- 当前立场
  presence TEXT, -- 存在状态
  linked_characters TEXT, -- 关联角色
  relationship_desc TEXT, -- 关系说明
  influence_on_romance TEXT -- 对恋爱线影响
);
```
