# 物品表

## sourceData.note

记录对主角有情感意义的非装扮物品。只收录能唤起回忆、承载心意或推动关系的物品。

【列定义】
- 列1: 物品名称 item_name（全表唯一，≤20字）
- 列2: 类型 item_type
- 列3: 数量 quantity（非负整数）
- 列4: 情感分量 emotional_weight
- 列5: 描述 description（≤80字）

【强制约束】
1.仅收录对主角或恋爱对象有情感意义、涉及关系记忆的物品。不收录纯功能杂物、装备、货币与点数。
2.同一物品情感意义升级时（如心意礼物转为定情信物），更新类型与描述，不新增行。
3.物品名称直接使用角色会说的口语化命名。
√ 他送的橘子糖
× 橘子味糖果
4.数量为非负整数，耗尽后可为0。
5.描述侧重物品的情感意义与回忆关联。

【字段取值】
类型：[定情信物, 纪念品, 心意礼物, 约会用品, 情绪物件]
  定情信物：明确承载双方感情承诺的物品，如戒指、情书信物、成对饰品，通常唯一。
  纪念品：与共同经历或关键事件绑定的物品，如电影票根、照片、旅行小物。
  心意礼物：单方赠送、尚未确认回应的礼物，情感回应待定。
  约会用品：为特定见面准备的功能物品，如野餐布、未送出的花束、活动门票。
  情绪物件：因当前情感状态而被赋予意义的日常物品，如同桌借过的橡皮、对方忘在桌上的水杯。
情感分量：[可有可无, 偶尔想起, 日常在意, 反复拿出来看, 随身携带, 刻进记忆]

## sourceData.initNode

按设定添加主角初始携带的物品。

## sourceData.deleteNode

禁止。

## sourceData.updateNode

已存在的物品，情感分量、类型或描述变化时更新。可堆叠物品不新建重复行。
SQL示例(消耗使用): UPDATE inventory SET quantity = quantity - 1 WHERE item_name = "艾莉丝给的橘子糖";
SQL示例(情感升级): UPDATE inventory SET item_type = "定情信物", emotional_weight = "随身携带", description = "原本只是她随手塞的糖，现在被主角穿绳挂在脖子上。糖纸上的折痕越来越深，每次紧张都会摸一下。" WHERE item_name = "艾莉丝给的橘子糖";

## sourceData.insertNode

物品名称不在表里时，新增一条。可堆叠物品不新建重复行。

SQL示例: INSERT INTO inventory (row_id, item_name, item_type, quantity, emotional_weight, description) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM inventory), "皱掉的电影票根", "纪念品", 1, "反复拿出来看", "第一次一起看电影的票根，被主角夹在手机壳里。边角已经起毛，背面有艾莉丝随手画的歪扭笑脸。");

## sourceData.ddl

```sql
CREATE TABLE inventory ( -- 物品表
  row_id INTEGER PRIMARY KEY, -- 行号
  item_name TEXT NOT NULL UNIQUE, -- 物品名称
  item_type TEXT NOT NULL CHECK(item_type IN ('定情信物', '纪念品', '心意礼物', '约会用品', '情绪物件')), -- 类型
  quantity INTEGER NOT NULL DEFAULT 1 CHECK(quantity >= 0), -- 数量
  emotional_weight TEXT NOT NULL, -- 情感分量
  description TEXT NOT NULL CHECK(LENGTH(description) <= 80) -- 描述
);
```
