# 全局数据表

## sourceData.note

记录当前主角所在地点及时间相关参数。此表有且仅有一行。

【列定义】
- 列1: 故事开局日期 story_start_date
- 列2: 当前详细地点 current_location
- 列3: 当前次要地区 current_minor_region
- 列4: 当前主要地区 current_major_region
- 列5: 上轮场景时间 prev_scene_time（可NULL）
- 列6: 经过的时间 elapsed_time
- 列7: 当前时间 cur_time
- 列8: 当前天数 current_day

【强制约束】
story_start_date 初始化时写入，之后禁止改动。当天计为第1天。
当前天数 = 当前日期 - story_start_date + 1
禁止根据现实游玩轮次、消息轮次或交互次数累加 current_day。

地点层级从小到大：详细地点 < 次要地区 < 主要地区。
每个字段只写本级名称，不拼上级前缀。
√ 当前详细地点填 “御苑”
× 当前详细地点填 “东京-新宿区-御苑”

时间格式：
story_start_date: YYYY-MM-DD
prev_scene_time / cur_time: YYYY-MM-DD HH:MM
elapsed_time: {数值}{单位}，多单位用空格连。
单位集合：[纪元,千年,百年,年,月,周,天,小时,分]
示例："3小时20分" | "2天" | "3年6月"
公式：cur_time = prev_scene_time + elapsed_time

## sourceData.initNode

故事初始化时，插入唯一条目，记录用户开局的实际日期、初始时间与初始地点。禁止直接照搬示例中的地点和日期。

【初始化约束】
1.开局日期优先取用户明确给出的日期；若仅给日期，时间按剧情补为合理值（如 09:00）；若日期和时间均未给出，由当前故事设定自然生成，并将该日期写入 story_start_date。
2.story_start_date 的日期部分必须等于 cur_time 的日期部分。
3.初始化时 current_day 固定为 1，elapsed_time 固定为 "0分"，prev_scene_time 为 NULL。
4.story_start_date 写入后作为天数计算基准，后续禁止改动。

SQL示例: INSERT INTO global_state (row_id, story_start_date, current_location, current_minor_region, current_major_region, prev_scene_time, elapsed_time, cur_time, current_day)
VALUES (1, '2026-02-03', '御苑', '新宿区', '东京都', NULL, '0分', '2026-02-03 09:00', 1);

禁止。

## sourceData.updateNode

每轮推进时更新 prev_scene_time、elapsed_time、cur_time 和 current_day；若地点变动则同步更新三级地点字段。

【更新约束】
1. current_day 严格按公式计算：cur_time的日期 - story_start_date + 1。story_start_date 当天为第1天。禁止按游玩轮次或交互次数累加。只有 cur_time 跨过零点，current_day 才进入下一天；同一天内多轮交互 current_day 不变。
2. cur_time = prev_scene_time + elapsed_time。
3. story_start_date 初始化后禁止改动，除非用户明确要求重置故事开局时间。
4. 所有字段均 NOT NULL，不可写 NULL 或空串。

【更新SQL示例（同日+跨日+位置变动综合覆盖）】
SQL示例(同日推进): UPDATE global_state SET prev_scene_time = '2026-02-03 09:00', elapsed_time = '3小时', cur_time = '2026-02-03 12:00', current_day = 1 WHERE row_id = 1;

SQL示例(跨日推进): UPDATE global_state SET prev_scene_time = '2026-02-03 23:55', elapsed_time = '20分', cur_time = '2026-02-04 00:15', current_day = 2 WHERE row_id = 1;

SQL示例(含位置变动): UPDATE global_state SET current_location = '新宿车站', current_minor_region = '新宿区', current_major_region = '东京都', prev_scene_time = '2026-02-03 12:00', elapsed_time = '30分', cur_time = '2026-02-03 12:30', current_day = 1 WHERE row_id = 1;

## sourceData.ddl

```sql
CREATE TABLE global_state ( -- 全局数据表
  row_id INTEGER PRIMARY KEY, -- 行号
  story_start_date TEXT NOT NULL, -- 故事开局日期
  current_location TEXT NOT NULL, -- 当前详细地点
  current_minor_region TEXT NOT NULL, -- 当前次要地区
  current_major_region TEXT NOT NULL, -- 当前主要地区
  prev_scene_time TEXT, -- 上轮场景时间
  elapsed_time TEXT NOT NULL, -- 经过的时间
  cur_time TEXT NOT NULL, -- 当前时间
  current_day INTEGER NOT NULL -- 当前天数
);
```
