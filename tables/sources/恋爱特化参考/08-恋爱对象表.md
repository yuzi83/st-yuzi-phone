# 恋爱对象表

## sourceData.note

记录对主角有恋爱关系或潜在恋爱关系的角色。

【列定义】
- 列1: 姓名 name（全表唯一）
- 列2: 性别 gender
- 列3: 年龄 age
- 列4: 一句话介绍 brief_intro（≤30字）
- 列5: 外貌特征 appearance（≤60字）
- 列6: 穿着打扮 outfit_text（≤40字）
- 列7: 基础属性 base_attributes
- 列8: 特有属性 special_attributes（可NULL）
- 列9: 所在地点 location_name
- 列10: 在场状态 presence_status（在场/离场）
- 列11: 人际关系 relation_state
- 列12: 对主角好感度 affection_state
- 列13: 对主角信任度 trust_state
- 列14: 对主角态度 attitude_to_protagonist
- 列15: 当前情绪 emotional_state
- 列16: 过往经历 past_experience（≤800字）
- 列17: 性爱次数 intimacy_count
- 列18: 交互选项 interaction_options

【强制约束】
1. 准入：仅录入有恋爱关系或潜在恋爱发展可能的角色。一次性路人、纯功能性NPC不录。
2. 职责边界：本表只记录恋爱对象对主角的关系阶段、好感、信任、当前态度与后续倾向。完整事件经过写入纪要表，角色主观记忆写入恋爱日记表。

3. brief_intro 仅允许客观事实类内容，严禁出现任何性格标签概述。
√ 经营城南杂货铺 / 曾救过主角的性命 / 艾莉丝的亲生兄长
× 性格孤僻的杂货铺老板 / 温柔且救过主角 / 偏执的艾莉丝兄长

4. outfit_text 仅写外在可见的服饰、饰品、妆容、持握物等可视元素，禁止写固有的长相特征、气质、气场等非视觉内容。
√ 深灰立领风衣配黑皮手套 / 左手无名指戴着一枚银色蛇纹戒
× 散发出高贵的气场、看起来像个军人

5. 所有情感类字段必须使用自然白话文，描述可观察的行为、神态、语气等迹象，保留不确定性和摇摆空间。禁止下定义、归纳状态、锁定结论。禁止量化数值。
√ 她看见主角会稍微多站一会儿，但嘴上照旧不饶人
√ 好像不讨厌待在一起，但也说不好是习惯还是别的
× 她对主角已形成稳定的依恋关系
× 处于好感上升期

6. interaction_options 必须同时遵循：
- 不使用第一人称"我"
- 动作发起方为主角，接收方为角色
- 必须是具体、有代入感的实际行动，避免干瘪的单词
√ 邀请他去约会 / 试探他的底线 / 假装没注意到他的目光
× 交谈 / 约会

【属性规则】
<属性规则>

</属性规则>

【字段取值】
人际关系：[陌生, 认识, 熟悉, 朋友, 暧昧, 心动, 交往, 热恋, 冷战, 稳定]
好感度：[排斥, 警惕, 好奇, 亲近, 在意, 心动, 依赖, 深爱, 矛盾]
信任度：[低, 中, 高]

以上取值非穷举，鼓励根据角色具体状态自行措辞。

## sourceData.initNode

为已登场的恋爱对象各插入一条（包含前任、已故对象等存在过恋爱关系或有恋爱发展可能的角色）。

## sourceData.deleteNode

仅在恋爱对象彻底转为非恋爱的重要角色，且不再有任何恋爱相关剧情与情感拉扯时，才可从本表删除并迁移至重要角色表。

【重点警告】恋人分手、死亡、失踪、冷战等情况，属于恋爱剧情的延续，绝对禁止删除该行！

SQL示例（彻底转为普通重要角色）: DELETE FROM romance_targets WHERE name = '艾莉丝';

## sourceData.updateNode

已存在的恋爱对象，任何字段变化时更新。关系阶段、好感度、信任度、态度、情绪变化时需及时更新。

【重点规则】恋人分手、死亡、失踪等情况只更新在场状态、人际关系、好感度等字段，绝不删除本行。

SQL示例(关系变化): UPDATE romance_targets SET relation_state = '主角:暧昧;神田:警惕', affection_state = '心动', trust_state = '高', attitude_to_protagonist = '愿意与主角独处并分享心事' WHERE name = '艾莉丝';
SQL示例(分手/死亡): UPDATE romance_targets SET relation_state = '分手', presence_status = '离场', emotional_state = '痛苦' WHERE name = '神谷真';

## sourceData.insertNode

新恋爱对象登场时新增一行。

SQL示例: INSERT INTO romance_targets (row_id, name, gender, age, brief_intro, appearance, outfit_text, base_attributes, special_attributes, location_name, presence_status, relation_state, affection_state, trust_state, attitude_to_protagonist, emotional_state, past_experience, intimacy_count, interaction_options) VALUES ((SELECT COALESCE(MAX(row_id), 0) + 1 FROM romance_targets), '神谷真', '男', 32, '地下情报贩子，对主角有兴趣', '身高180cm，左眼有疤', '黑西装', '力量:47; 敏捷:58; 体质:49; 智力:66; 感知:70; 魅力:54', '伪装:74', '旧港仓区', '离场', '主角:若即若离', '好奇', '低', '对主角保持观察但保持距离', '试探', '长期向多方出售消息，对主角的身份产生兴趣。', 0, '试探他的底线');

## sourceData.ddl

```sql
CREATE TABLE romance_targets ( -- 恋爱对象表
  row_id INTEGER PRIMARY KEY, -- 行号
  name TEXT UNIQUE, -- 姓名
  gender TEXT, -- 性别
  age INTEGER, -- 年龄
  brief_intro TEXT, -- 一句话介绍
  appearance TEXT, -- 外貌特征
  outfit_text TEXT, -- 穿着打扮
  base_attributes TEXT, -- 基础属性
  special_attributes TEXT, -- 特有属性
  location_name TEXT, -- 所在地点
  presence_status TEXT, -- 在场状态
  relation_state TEXT, -- 人际关系
  affection_state TEXT, -- 对主角好感度
  trust_state TEXT, -- 对主角信任度
  attitude_to_protagonist TEXT, -- 对主角态度
  emotional_state TEXT, -- 当前情绪
  past_experience TEXT, -- 过往经历
  intimacy_count INTEGER, -- 性爱次数
  interaction_options TEXT -- 交互选项
);
```
