# 消息记录表

## sourceData.note

记录已实际发生的私聊消息。一条聊天气泡一行；多条连发必须拆多行。只写气泡文字，不写旁白、动作、心理、剧情总结或多条拼接。

# 私聊字段规则
- 私聊对象固定为 NPC：会话标题 = 聊天对象 = 该 NPC 名；禁止写 user、玩家、主角、我。
- 用户消息：发送者写“主角”或“我”，发送者身份=user。
- NPC 消息：发送者写 NPC 名，发送者身份=assistant。
- 系统提示仅在必须显示时使用：发送者=系统，发送者身份=system，会话标题/聊天对象仍写该 NPC 名。
- 同一私聊线程保持同一会话ID；同一 NPC 后续新话题可接续原会话。
- 请求ID每条唯一；同次回复可用 base_reply_1、base_reply_2 等后缀。
- 无图片/视频写 none；回复某条消息时，回复到消息ID填被回复请求ID，否则留空。

# 会话拓扑强约束
- 每个会话只能表示“主角 ↔ 一个 NPC”的私聊。禁止创建 NPC↔NPC、群聊、多人聊天、旁观聊天。
- chat_target 永远是正在和主角聊天的单个 NPC。
- sender_role=user 时，sender 只能是“主角”或“我”。
- sender_role=assistant 时，sender 必须等于 chat_target。
- 如果某 NPC 转述另一个 NPC 的内容，也仍然是该 NPC 发给主角的消息，不得把另一个 NPC 写成发送者。
- 不确定是否是主角参与的私聊时，不插入。

# 聊天文风
写成角色真的会发出的微信气泡：符合人设、关系距离、口癖、情绪和当前处境。可短句、断句、补发、嘴硬、试探、犹豫、少量表情；禁止小说腔、翻译腔、客套模板、舞台动作、内心独白、总结陈词和长篇说明。

# 主动私聊
可让重要 NPC 主动联系主角，但必须有关系或事件动机。优先选择：恋爱对象、重要角色、刚离场/不在现场却与当前事件有关的人。避免连续多轮只让同一两个联系人主动；也不要让无名路人、纯背景角色、当前正面对面的人无理由发微信。

# 最低正确性
消息记录会直接影响用户对角色关系和会话归属的判断。字段归属错比文风差更严重。输出前必须确认会话对象、发送者、sender_role 三者一致。若不确定私聊对象，宁可少插入，不要把主角写成聊天对象，也不要生成 NPC↔NPC 私聊。

## sourceData.initNode

故事开始时可插入少量已成立的初始私聊。每行一条气泡；会话标题=聊天对象=NPC名；用户消息发送者为“主角/我”，NPC消息发送者必须等于聊天对象。没有明确初始聊天时不必硬填。禁止用本表记录 NPC↔NPC 或多人聊天。

## sourceData.deleteNode

默认不删除历史消息。仅删除明显重复、误把旁白/总结写成消息、字段错位严重且无法安全修正的错误行。不要为了刷新列表删除正常聊天。

## sourceData.updateNode

默认不更新旧消息正文。仅修正明显字段错误、会话归属错误、发送者身份错误、媒体描述误填或请求ID/回复ID错误。修正时必须保持会话标题=聊天对象=单个NPC；sender_role=assistant 时 sender 必须等于聊天对象。剧情补充、解释、新话题一律新增消息行。

## sourceData.insertNode

有新的私聊气泡成立时插入，每次通常1-4行。先确定正在和主角聊天的单个 NPC，再强制保持：会话标题=聊天对象=NPC名。用户消息用 sender_role=user 且 sender 只能是“主角/我”；NPC消息用 sender_role=assistant 且 sender 必须等于聊天对象。一条气泡一行，NPC连发要拆行。

NPC可主动发消息给主角，但只限重要且有动机的角色：恋爱对象、重要角色、刚离场角色、与当前事件有关且可能联系主角的人。若近期总是同一联系人主动，优先从其他有动机的重要NPC中选择；无动机则不强行新增。不要把两个 NPC 之间的联系写进本表。

## sourceData.ddl

```sql
CREATE TABLE message_records (
  row_id INTEGER PRIMARY KEY, -- 行号
  session_id TEXT, -- 会话ID
  session_title TEXT, -- 会话标题
  sender TEXT, -- 发送者
  sender_role TEXT, -- 发送者身份
  chat_target TEXT, -- 聊天对象
  content TEXT, -- 消息内容
  send_time TEXT, -- 消息发送时间
  image_desc TEXT, -- 图片描述
  video_desc TEXT, -- 视频描述
  request_id TEXT, -- 请求ID
  reply_to_id TEXT -- 回复到消息ID
);
```
