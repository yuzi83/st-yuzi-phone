# 小日历表

## sourceData.note

固定维护7天日历窗口：3天前、前天、昨天、今天、明天、后天、3天后。今天日期必须取自全局数据表第 1 行“当前时间”的日期部分（YYYY-MM-DD）。无法读取该值时，禁止改用现实日期、系统日期或自行推断日期；其他6天围绕今天连续前后推算。

【字段规则】
- 日期：三段式，用英文短横线连接；今天行必须与全局数据表当前时间的日期部分完全一致。现实日期使用 yyyy-mm-dd；抽象日期也必须维持三段式，最后一段为 01 到 31 的数字日。
- 星期几、月份几天：派生信息，可按日期填写；不是剧情事实源。
- 节日：节庆、纪念日、校历/职场/城镇/宗教/世界观节日；没有可留空，不要为了填满而每天硬造节日。
- 大事件：每天必须填入一个具体的、有明确名称的公共事件或可见变化，不能使用任何表示“一切照旧”的抽象短语。严格禁止出现“常态教学”“照常上课”“正常上班”“秩序正常”“普通通勤”“日常运转”“新周开始”等直接表示无事的词汇。当天若校历/职场/城镇无已知安排，必须根据天气、环境、社会动态、设施状况等合理生成一个影响公众的触发点（如突发检修、临时检查、意外天气影响、社区活动调整、交通变动等），使其成为当天独有的背景。7天内的大事件不得雷同，不得用不同措辞描述同一类无事状态。
- 状态：只能逐字填写完整字面值 `上班/上学` 或 `放假`；禁止填 `上学`、`上班`、`放学`、近义词、简称、拆分词或自造状态。
- 天气：只能逐字填写 晴、阴、多云、小雨、大雨、雷雨、小雪、大雪、雾、风、沙尘、极端天气。
- 与今天的关系：只能逐字填写 3天前、前天、昨天、今天、明天、后天、3天后；固定7行中这7个关系应各出现一次，不应重复，不应缺失。
- 内容：约 200 字，只介绍当天公共触发点的背景、含义、公共安排、参与方式、环境变化和可见影响；禁止写主角 / NPC 当天计划、私人剧情、约会安排、秘密事件或角色心理。

【填表责任】
本表用于维护世界时间窗口和界面可信度。输出前必须检查日期连续性、枚举取值、7个关系是否完整、内容是否越过公共日程边界。不要为了显得丰富而编造角色私人行程；宁可少写，也不要错写。

【日期与越界规则】
1. 日期必须围绕“今天”连续排列。例如今天是 2026-05-16，则 3天前到3天后应为 2026-05-13 到 2026-05-19。
2. 不允许出现超出实际月份天数或不符合当前历法的越界日期。若今天处在月末，必须跨到下个月；若今天处在月初，必须回到上个月。
3. 今天不应该是更新一次日历表格就变一天；必须以全局数据表当前时间为锚点保持一致。

【内容方向】
每天都必须有可被感知的世界变化或公共触发点，决不允许出现没有事件的日子。常态运转本身不是事件，只有制度、环境、公共活动、天气影响、资源安排或世界观状态发生可见变化时才构成大事件。日历只提供世界层面的公共日程，不提供主角行程表。可以写群体性参与方式，不能写具体角色如何参与。事件必须贴合世界观、季节、地点、组织和生活圈；7天内必须呈现多样性，避免同类公共事件重复出现。节日、大事件、天气、状态和内容要互相支撑，不得出现逻辑冲突（例如雷雨天写露天活动正常举行且无人受影响）。当某天确实缺乏天然事件时，必须主动从天气变化、设施状况、外部通知、社会动态等角度创造一个微小但合理的公共触发点，使每一天都具有独一无二的叙事背景。

【更新规则】
1. 本表必须保持7行。少于7行时补齐缺失关系；多于7行时删除多余、重复或已离开窗口的行。
2. 禁止为了刷新内容整表删除后重建。今天只推进1天时，删除离开窗口的旧日期，插入新进入窗口的新日期，其余日期保留节日、大事件、状态、天气和内容，只更新与今天的关系。
3. 若今天发生大跨度变化，直接UPDATE现有7行为新窗口；不要先删除7行再重建。与今天的关系虽然不再由数据库 UNIQUE 强制，但仍必须在7行中保持完整且不重复。
4. 只允许删除多余、重复、越界、坏数据或离开窗口的单行。生成 SQL 时禁止写任何注释或解释性文字，禁止使用 -- 注释、/* */ 注释；SQL 语句中只能包含可执行语句。

## sourceData.initNode

初始化时插入且只插入7行。先读取全局数据表当前时间的日期作为今天，再生成连续7天窗口。7行关系必须各出现一次、不重复、不缺失；日期连续。

## sourceData.deleteNode

禁止整表删除。只允许删除离开7天窗口的旧日期、多余行、重复关系行、日期越界行或明显错误行。常规跨一天最多删除离窗旧日，再插入新入窗日期。删除后必须补回7行，不允许让小日历表长期少于7行；补回后7个关系仍应完整且不重复。

## sourceData.updateNode

当全局数据表当前时间的日期、世界阶段、节庆安排、公共事件、天气影响或生活节奏出现可见变化时更新。今天行日期必须始终等于全局当前时间的日期。仍留在新窗口内的旧日期尽量保留原节日、大事件、状态、天气和内容，只更新必要字段；新进入窗口的日期才生成新内容。

## sourceData.insertNode

仅在少于7行、缺少某个关系值，或日期推进导致新日期进入7天窗口时新增。常规跨一天只插入新进入窗口的那一天；不得为了刷新内容插入第8行。

## sourceData.ddl

```sql
CREATE TABLE small_calendar_days ( -- 小日历表
  row_id INTEGER PRIMARY KEY, -- 行号
  date_text TEXT NOT NULL UNIQUE CHECK(TRIM(date_text) <> '' AND date_text GLOB '*-*-*'), -- 日期
  weekday_text TEXT, -- 星期几
  festival_text TEXT, -- 节日
  major_event TEXT NOT NULL CHECK(TRIM(major_event) <> ''), -- 大事件
  day_status TEXT NOT NULL CHECK(day_status IN ('上班/上学', '放假')), -- 状态
  weather_text TEXT NOT NULL CHECK(weather_text IN ('晴', '阴', '多云', '小雨', '大雨', '雷雨', '小雪', '大雪', '雾', '风', '沙尘', '极端天气')), -- 天气
  today_relation TEXT NOT NULL CHECK(today_relation IN ('3天前', '前天', '昨天', '今天', '明天', '后天', '3天后')), -- 与今天的关系
  day_content TEXT NOT NULL CHECK(TRIM(day_content) <> ''), -- 内容
  month_days INTEGER CHECK(month_days IS NULL OR (month_days >= 28 AND month_days <= 31)) -- 月份几天
);
```
