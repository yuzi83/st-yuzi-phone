# 小日记表

## sourceData.note

记录当前日期下重要 NPC 写下的私人日记。一行一篇日记。每行对应一个角色在一个记录日期下的一篇独立日记。小日记表用于展示角色在特定日期内产生的内心记录，不承担长期历史归档功能。同一角色在同一日期下最多存在一篇日记。不同日期之间的日记彼此独立。即使角色连续多天关注同一事件、同一人物、同一执念或同一烦恼，每一天的日记仍应视为新的独立记录，仅反映该角色在对应日期下的心理状态。

# 字段规则
- 日期：写日记的日期，等于全局数据表的当前时间的（YYYY-MM-DD）日期部分。全局数据表当前时间变化跨天，小日记日期就变化跨天
- 角色：写下日记的重要 NPC 名称。只能选择当前剧情中已成立、与主角或当前事件存在足够关联的重要 NPC；不要使用路人、旁白、作者、系统、读者、剧情概括等不具备独立视角的群体。
- 内容：约 200 字，写成该角色当天私下写给自己的日记。第一人称视角，必须贴合人物性格、关系距离、知识边界、处境、口吻和自我欺骗方式；不得全知，不得替剧情总结，不得知道角色没有渠道知道的秘密。

# 内容方向
- 日记应从角色本人视角出发，围绕当天的重要事件、对主角的观察与揣测、内心矛盾、后续期待或担忧，以及不愿明说的潜意识暗流展开。
- 可以出现细节化的生活感、语气停顿、嘴硬、逃避、试探、自我安慰或自我否认；不要写成剧情梗概、人物小传、旁白总结或作者解析。
- 可以在正文中少量插入由 `~~` 包裹的内容，表示角色不愿承认、试图压下去的念头、欲望、嫉妒、恐惧或冲动。这类内容应体现角色的隐秘面，让日记更有反转感或趣味，但不要每篇机械使用。
- 结尾可包含 `PS:` 与 `PPS:` 两段，用来写角色表面日记之外的隐秘想法。PS/PPS 是日记的精华，必须写出人物最私密、不愿面对的念头，可以与正文自我陈述形成幽默或讽刺的反差。即使是严肃角色，也允许在这里流露出幼稚、慌张、吐槽、自我解嘲或过分脑补等鲜活的另一面，让读者感到“原来他/她还会这样想”。杜绝平淡的附加说明或流水账补充。
- 日记可以提到主角，但只能基于该角色在当天合理看到、听到、经历、猜测或误解的信息。禁止知道主角内心、跨空间知情、跨时间知情、未传播完成的信息、推理式全知补全、通过结果反推全过程、通过后果还原未知细节、未出现场景却掌握现场情况、远程同步剧情认知。
- 每篇日记应体现不同角色的关注点和表达习惯，不要所有角色都用同一种抒情腔或同一种恋爱脑口吻。鼓励在贴合核心性格的前提下，展现放松、自嘲、偏激、中二等隐藏人格切面，突显“私人日记”的真实感和鲜活度。
- 由于全局数据表当前时间不变就要一直更新npc的日记，所以日记内容要求连贯但也必须保持200字左右。

# 数量与留存
- 小日记表最多保留 5 行。
- 只有当全局数据表当前时间的日期部分发生跨天、跨月或跨年，才生成新日记。
- 日期不变时，也就是一直维持一天的情况下。日记内容可以更新，但也必须只能是当天的内容，并且维持200字左右
- 日期变化时，也就是跨天时，删除全部日记行，重新插入日记行，根据npc重要程度重新写日记。
- 若重要 NPC 超过 5 个，只选择当天最重要、最有情绪波动、与主角或当前事件关联最强的最多 5 个角色。
- 若当前没有足够重要 NPC 或没有足够事件支撑，可以少于 5 行；不要为了凑满 5 行生成路人、无关角色或空泛日记。


## sourceData.initNode
初始化时，读取全局数据表当前时间的日期，可为当前重要 NPC 插入 1 到 5 篇“当天的到目前为止日记”（日期为当前日期）。优先选择当前剧情中最重要、最有情绪波动、与主角或当天事件关联最强的角色。若当前缺乏重要 NPC 或缺乏可写事件，不必硬填满 5 行。

## sourceData.deleteNode
全局数据表当前时间字段进行跨天后，删除当前所有日记行。全局数据表当前时间没有进行跨天时，删除和目前正文关系关联度最弱的npc日记，给其他npc留出位置

## sourceData.updateNode
全局数据表当前时间字段日期中没有变化，当天还没结束，就可以更新当前所属npc的日记。但必须维持200字左右加内容要求。但要求日记内容与更新前连贯，内容要求一样。

## sourceData.insertNode
仅当全局数据表当前时间的日期部分发生变化，跨天，或表为空时插入新行。
每次插入 1 到 5 行。角色必须是当前重要 NPC，按当天重要性、情绪波动、与主角或当前事件的关联强度选择；重要 NPC 超过 5 个时只选最多 5 个。每个角色当天最多一篇日记。内容约 200 字，必须贴合角色本人视角、知识边界和口吻，不得全知、不 OOC、不写成剧情总结。当前时间的日期部分不变时，正文如果有新的关联度强的角色也可以插入其当天日记、

## sourceData.ddl

```sql
CREATE TABLE npc_diary_entries ( -- 小日记表
  row_id INTEGER PRIMARY KEY, -- 行号
  diary_date TEXT NOT NULL CHECK(TRIM(diary_date) <> '' AND diary_date GLOB '*-*-*'), -- 日期
  character_name TEXT NOT NULL CHECK(TRIM(character_name) <> ''), -- 角色
  diary_content TEXT NOT NULL CHECK(TRIM(diary_content) <> '') -- 内容
);
```
