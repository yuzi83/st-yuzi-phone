# 小日记表

## sourceData.note

记录当前日期下重要 NPC 写下的私人日记。一行一篇日记。

【字段规则】
- 日期：写日记的日期应为“跨天发生前的旧日期”，即刚结束的那一天，而非当前时间日期。例如：当前时间从 2026-05-11 23:00 变为 2026-05-12 01:00，日记日期必须为 2026-05-11。
- 角色：写下日记的重要 NPC 名称。只能选择当前剧情中已成立、与主角或当前事件存在足够关联的重要 NPC；不要使用路人、旁白、作者、系统、读者或不具备独立视角的群体。
- 内容：约 200 字，写成该角色当天私下写给自己的日记。必须贴合人物性格、关系距离、知识边界、处境、口吻和自我欺骗方式；不得全知，不得替剧情总结，不得知道角色没有渠道知道的秘密。

【内容方向】
- 日记应从角色本人视角出发，围绕“刚过去的这一天”中的重要事件、对主角的观察与揣测、内心矛盾、后续期待或担忧，以及不愿明说的潜意识暗流展开。
- 可以出现细节化的生活感、语气停顿、嘴硬、逃避、试探、自我安慰或自我否认；不要写成剧情梗概、人物小传、旁白总结或作者解析。
- 可以在正文中少量插入由 `~~` 包裹的内容，表示角色不愿承认、试图压下去的念头、欲望、嫉妒、恐惧或冲动。这类内容应体现角色的隐秘面，让日记更有反转感或趣味，但不要每篇机械使用。
- 结尾可包含 `PS:` 与 `PPS:` 两段，用来写角色表面日记之外的隐秘想法。PS/PPS 是日记的精华，必须写出人物最私密、不愿面对的念头，可以与正文自我陈述形成幽默或讽刺的反差。即使是严肃角色，也允许在这里流露出幼稚、慌张、吐槽、自我解嘲或过分脑补等鲜活的另一面，让读者感到“原来他/她还会这样想”。杜绝平淡的附加说明或流水账补充。
- 日记可以提到主角，但只能基于该角色在“刚过去那一天”合理看到、听到、经历、猜测或误解的信息。禁止知道主角内心、未公开行动、封闭场景细节或其他角色隐私。
- 每篇日记应体现不同角色的关注点和表达习惯，不要所有角色都用同一种抒情腔或同一种恋爱脑口吻。鼓励在贴合核心性格的前提下，展现放松、自嘲、偏激、中二等隐藏人格切面，突显“私人日记”的真实感和鲜活度。

【数量与留存】
- 小日记表最多保留 5 行。
- 只有当全局数据表当前时间的日期部分发生跨天、跨月或跨年，才生成新日记。
- 日期变化时，为“变化前的完整旧日期”生成一篇日记。若一次跨越多天，只选择离新日期最近的那个完整过去日（即刚过去的最后一天）生成日记；不要为中间每一个空白日期补日记，也不要为变化后的新日期生成空洞日记。
- 生成新日记前，先删除表中所有日期不等于本次日记日期（即旧日期）的条目，再插入新行；不要更新旧日记正文。
- 若重要 NPC 超过 5 个，只选择当天最重要、最有情绪波动、与主角或当前事件关联最强的最多 5 个角色。
- 若当前没有足够重要 NPC 或没有足够事件支撑，可以少于 5 行；不要为了凑满 5 行生成路人、无关角色或空泛日记。

【最低正确性】
小日记是角色主观记录，不是全局真相表。输出前必须检查：
- 日期是否为“跨天发生前的旧日期”（刚过去的那一天），而非当前时间日期；
- 角色是否是重要 NPC；
- 内容是否符合该角色已知信息，是否存在全知视角；
- 是否错误更新旧日记；
- 是否超过 5 行。

## sourceData.initNode
初始化时，如果能读取全局数据表当前时间的日期，可为当前重要 NPC 插入 1 到 5 篇“当天的到目前为止日记”（日期为当前日期）。优先选择当前剧情中最重要、最有情绪波动、与主角或当天事件关联最强的角色。若当前缺乏重要 NPC 或缺乏可写事件，不必硬填满 5 行。

## sourceData.deleteNode
当本表存在旧日期日记、超过 5 行、重复角色日记、路人日记、全知视角日记、明显 OOC 日记或坏数据时删除。日期变化触发新日记生成时，应删除所有日期不等于本次应写入的“旧日期”的日记；不要保留历史日记。禁止为了重写同一天内容而删除并重建有效日记。

## sourceData.updateNode
一般情况下禁止更新已有日记正文。

## sourceData.insertNode
仅当全局数据表当前时间的日期部分发生变化，或表为空时插入新行。
插入前先确定本次应写入的“旧日期”：若因跨天触发，旧日期为变化前的日期；若表为空初始化，旧日期为当前日期。
若表中存在日期不等于该“旧日期”的日记，先全部删除，再插入该日期的新日记。
每次插入 1 到 5 行。角色必须是当前重要 NPC，按当天重要性、情绪波动、与主角或当前事件的关联强度选择；重要 NPC 超过 5 个时只选最多 5 个。每个角色当天最多一篇日记。内容约 200 字，必须贴合角色本人视角、知识边界和口吻，不得全知、不 OOC、不写成剧情总结。

## sourceData.ddl

```sql
CREATE TABLE npc_diary_entries ( -- 小日记表
  row_id INTEGER PRIMARY KEY, -- 行号
  diary_date TEXT NOT NULL CHECK(TRIM(diary_date) <> '' AND diary_date GLOB '*-*-*'), -- 日期
  character_name TEXT NOT NULL CHECK(TRIM(character_name) <> ''), -- 角色
  diary_content TEXT NOT NULL CHECK(TRIM(diary_content) <> '') -- 内容
);
```
