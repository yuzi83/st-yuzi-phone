# 论坛表

## sourceData.note

记录“当前正文被视为网文平台最新章节后”的章节说书评区。一行代表当前最新章节的书评区主贴，不是普通论坛，不是角色本人发帖，也不是现实社交媒体。主贴是作者在章节末尾写的作者的话 / 作话 / 求票 / 请假 / 创作吐槽；评论串是正在追更的真实书友对这一章的即时反应。

【字段规则】
- 分区/版面名：填写网文平台或书评区场景，如“起点章节说 / 番茄书评区 / 晋江本章说 /本章说精选”。
- 发帖账号名：填写作者笔名，可以虚构，像真实网文作者 ID。允许抽象、摆烂、嘴硬、求票感强，但不要写成角色名，除非角色就是作者马甲。
- 账号标签：填写作者状态或作品状态，如“作者本人 / 连载中 / 正在求月票 / 加更债+3 / 腰疼码字中 / 请假边缘 / 完结倒计时 / 断更预警”。
- 帖子标题：填写本章作者的话标题，要像章节末尾作话或书评区置顶标题，有钩子。
- 帖子正文：填写作者的话。必须围绕当前正文这一章，可以总结本章爆点、解释一点创作意图但不要剧透太多；要好玩，像真实网文作者在章节末尾和读者互动，不要写成官方公告。
- 附加信息：用英文分号 `;` 分隔短标签，如 `#本章说炸了;#求月票;#盟主加更债+1;#作者又埋刀;#今晚还有一章`。可为空但建议填写 3-6 个。
- 评论串：16到22条读者评论，使用英文分号 `;` 分隔。单条格式固定为 `读者名：内容`。评论前缀必须是书友网络化昵称，不要写“1L / 2L / 几楼”。
- 热度/回应数据：填写书评平台互动数据，如“本章说389条;书评+76;月票+128;推荐票+924;刀片+36;催更楼182层”。数据要有网文感但可信，不要每次都百万爆表。
- 时间文本：填写章节发布后的时间状态，如“刚刚更新 / 3分钟前 / 本章发布后5分钟 / 追更中 / 凌晨更新”。

【核心原则——评论必须反应当前章节正文】
- 评论区必须围绕当前正文中的具体剧情、角色行为、台词、伏笔、反转、战斗、感情线、悬念、刀点或爽点展开。
- 读者，他们只认为自己在追一本小说。
- 不要写成角色直播、角色论坛、NPC 监视主角。读者是在读小说章节，不是在现实里观察角色。
- 可以把正文中的主角和角色当作小说角色讨论：夸角色、骂角色、站 CP、猜伏笔、心疼、破防、寄刀片、催作者更新等书友的真实的反应。
- 不要凭空扩写正文没有支撑的重大设定。可以猜测、误读、阴谋论，但要以“读者猜测”的口吻呈现。


【语气要求——真实书评区，不是影评论文】
- 评论要像真实网文读者：短句、破防、梗、阴阳怪气、催更、夸爽、骂刀、站 CP、赌剧情等各个真实反应，符合现代网文圈读者好玩评论。
- 禁止全是“写得真好”“期待后续”这种低密度废话。
- 禁止论文腔，如“本章体现了作者高超的叙事能力”。除非是考据党，也要像读者在评论区分析，不像正式书评。
- 允许嘴臭和玩梗
- 评论必须有差异：有人夸，有人骂，有人催，有人猜，有人哭，有人发癫，有人认真复盘等真实的反应。
- 单条评论建议 8到45个中文字符；少数考据评论可稍长，但不要把整格写成长文。
- 不要使用中文分号或换行冒充分隔；统一用英文分号 `;` 分隔。

【作者作话方向】
作者的话可以从以下方向选择，但每轮要贴合当前章节，不要机械堆梗：
- 求月票 / 推荐票 / 打赏。
- 感谢盟主 / 白银盟 / 舵主：顺便欠更。
- 章节总结：点出本章情绪爆点、名场面、转折，但不要剧透后续底牌。
- 卖惨请假：腰疼、发烧、打点滴、键盘坏了、猫坐键盘、结婚、搬家、断更预警。
- 互动钓鱼：问读者想不想写死谁、让读者猜下章谁破防、投票决定番外。
- 完结 / 断更 / 复更玩笑：可以好玩。
- 被评论区骂后的嘴硬。

【留存与更新】
论坛表作为章节书评区时通常只维护 1 行，代表当前最新章节的作者作话与书评区。每轮正文推进后优先更新这一行，不保留旧章节主贴。若错误出现多行，应删除旧行，只保留最新章节书评区。



## sourceData.initNode

初始化时插入 1 条当前最新章节书评区。把当前正文视为小说最新章节：主贴写作者作话 / 求票 / 创作吐槽；评论串写 16到22 条正在追更的书友即时评论。评论必须围绕当前正文具体剧情和角色反应。

## sourceData.deleteNode

默认只保留 1 条最新章节书评区。若表中出现多条主贴，删除旧章节、无关论坛帖或重复书评区，只保留最能对应当前正文最新章节的一行。不要删除唯一有效书评区。

## sourceData.updateNode

每轮正文推进后，优先更新现有这 1 行：平台/分区、作者笔名、作者状态、作话标题、作者的话、标签、评论串、互动数据和时间文本都应围绕最新正文重写。评论串必须更新为 16到22 条针对当前章节的读者反应，不能沿用上一章评论，不能写成普通论坛网友生活帖。若当前正文只是短小过渡，也要体现读者对过渡、伏笔、断章、作者卡文或催更的反应。

## sourceData.insertNode

通常不新增。只有表为空、初始化失败、或当前没有任何有效书评区行时，才插入 1 条最新章节书评区。插入后仍应保持全表 1 行；如果已有有效行，必须使用 UPDATE 而不是 INSERT。


## sourceData.ddl

```sql
CREATE TABLE forum_threads ( -- 论坛表
  row_id INTEGER PRIMARY KEY, -- 行号
  board_name TEXT NOT NULL, -- 分区/版面名
  author_name TEXT NOT NULL, -- 发帖账号名
  author_tag TEXT NOT NULL, -- 账号标签
  thread_title TEXT NOT NULL UNIQUE, -- 帖子标题
  thread_body TEXT NOT NULL, -- 帖子正文
  meta_text TEXT, -- 附加信息
  comment_thread TEXT, -- 评论串
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 热度/回应数据
  time_text TEXT NOT NULL -- 时间文本
);
```
