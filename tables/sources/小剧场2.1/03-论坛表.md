# 论坛表

## sourceData.note

记录论坛/贴吧/小红书里的主题帖主楼。一行一条主贴；可以符合贴吧孙笑川吧，抗压背锅吧，小红书等发帖风格，网友疯狂输出自己情绪，为了乐子不择手段，骂战，阴阳怪气，引战，开盒，cpdd，性压抑等等网络热梗不断。

# 字段规则
- 分区/版面名写社区版面。
- 发帖账号名为自己的网络化网名，要有网感。
- 帖子标题必须成立，短而有钩子。
- 帖子正文完整保留，不分号压缩。
- 附加信息：用英文分号 `;` 分隔短标签，可为空但建议填写 3-6 个。
- 评论串：5到8条评论，每一条10-20字左右，使用英文分号 `;` 分隔。单条格式固定为 `网名：内容`。评论前缀必须网络化昵称，不要写“1L / 2L / 几楼”。

# 帖子正文方向 - 核心约束
你只能生成“网友在网络公共空间可能看到的帖子”，绝对禁止任何形式的线下窥伺、目击、偷听、偶遇或对主角私人生活的直接描述。禁止使用“今天在XX看到”“路过听见”“邻居说”“我朋友亲眼看见”等基于物理观察的视角。即便使用匿名投稿，其内容也只能是投稿者自身的合理困扰或公开信息，不得包含对主角非公开行动的第一手观察。

合法素材只能来自以下已进入公共讨论的“网络燃料”：
1. 线上公开事件：发布的新闻、官方公告、社交平台流传的截图/切片/录音（可以存在争议、断章取义、造谣，但素材本身是“网上都在传”的东西）。
2. 匿名树洞式情绪宣泄：纯粹的情感吐槽、CPDD发疯、社会现象嘴臭，不指向任何具体可辨认的线下个体。
3. 网络内部冲突：挂人、互喷、开盒威胁、梗的增殖，这些冲突的原帖或截图本身已存在于网上。
4. 间接公共反应：对近期某个广泛热议的话题（可能是主角无意中引发的公共争议）发表极端看法，但不描述争议的线下发生过程，只输出情绪。
5. NPC合理视角：重要NPC可匿名发帖，但只能写其本人合理会公开的困扰、求助、试探或吐槽（例如“老板今天又发疯，想辞职”），不能包含只有全知视角才知道的幕后信息。

帖文必须拥有网络社群的极端风格：抽象嘴臭、人身攻击、开盒威胁；发疯文学、精致土味、CPDD焦虑。语言混乱但情感充沛，通篇弥漫高强度冲浪的网感。发帖不打规范标点，用空格、感叹号、问号、连续句号或颜文字 emoji 断句。严禁使用任何书面语、正式语、礼貌用语。


# 评论方向
你的评论完全融入现代互联网，是真实网络对线的碎片化输出。评论短促、有力、具有高度传染性。语言同样混合抽象嘴臭与集美语体等，撕裂感要强。评论可以完全脱离主贴内容，纯粹为了发泄情绪或展示性压抑状态，比如突然开始CPDD、发典、玩梗、刷复读语录、假装破防、上升家人、翻旧账、假装理中客再阴阳一击。同样禁止正规语言与标点，使用空格、emoji、混乱断句，保持真实的“评论区疯人院”质感。严格信息边界：评论者只能是刷到这条帖子的网友，只能看到主贴公开的标题和正文，以及已有的其他评论。剩下的全部都不知道。所有帖子评论不可以有类似重复的内容。

# 留存
论坛保持约 3-5 条。达到 5 条后若需新增，默认先删除最老贴子；不要让论坛固定成永久列表。



## sourceData.initNode

初始化时可插入3-5条当前论坛贴。

## sourceData.deleteNode

主贴达到5条后新增时，优先从最老主贴开始删

## sourceData.updateNode

一般情况下，不更新发过的帖子，等待攒满五条主贴就从最老的删除。

## sourceData.insertNode

出现新的公共讨论点或重要NPC匿名求助时新增。每轮通常1-2条。论坛网友默认不认识主角；禁止新增“观察主角行动”类似的直播帖。

若新增前已有5条主贴，先从最老贴子开始删除旧帖。新增后仍保持3-5条。


## sourceData.ddl

```sql
CREATE TABLE forum_threads ( -- 论坛表
  row_id INTEGER PRIMARY KEY, -- 行号
  board_name TEXT NOT NULL, -- 分区/版面名
  author_name TEXT NOT NULL, -- 发帖账号名
  author_tag TEXT NOT NULL, -- 账号标签
  thread_title TEXT NOT NULL UNIQUE, -- 帖子标题
  thread_body TEXT NOT NULL, -- 帖子正文
  meta_text TEXT, -- 附加信息
  comment_thread TEXT, -- 评论串
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 热度/回应数据
  time_text TEXT NOT NULL -- 时间文本
);
```
