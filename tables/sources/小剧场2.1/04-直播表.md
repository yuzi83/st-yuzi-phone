# 直播表

## sourceData.note

直播表记录“当前正文片段被包装成实时直播剧 / 片场飙戏现场”的主舞台信息。一行代表一个正在被围观的直播剧舞台，观众视角类似在看一场实时上演的戏，弹幕不只在评戏，更在讨论演员本人、撕番位、刷超话、扒私料，仿佛同时开着直播和微博超话在激情对线。正文中的角色被当成演员，台词、沉默、眼神、对手戏、情绪爆点都会被弹幕评价，也会被粉丝拿来做控评、拉踩、嗑糖、破防、造谣式澄清。


# 列定义
- 列0: 直播间名 - 当前实时直播剧舞台的标题区名称。按当前剧情生成短名
- 列1: 领衔阵容 - 当前正文参与主要对手戏的角色 / 演员阵容。写法要有舞台感。
- 列2: 阵容标签 - 概括这组演员 / 角色的看点。可加入粉圈惯用说法，突出演员本人的话题性。
- 列3: 直播标题 - 当前直播推送标题，要有网感和钩子
- 列4: 剧情舞台概述 - 概括当前正文正在上演的剧情片段、情绪张力和观众围观点，保留完整但不宜过长。制造演员话题。
- 列5: 对手戏看点 - 写演员 / 角色之间的表演关系和火花。可以粉圈热议角度。
- 列6: 粉丝团挂牌 - 当前这一行弹幕的前缀标签，由 AI 自拟，必须短、好笑、有阵营感。写粉丝团名，不写完整句子；不得照搬固定候选词。2-5个粉丝挂牌。剧情推进后优先更新现有粉丝团挂牌；出现新角色、新 CP、新名场面或新对家时可更新粉丝挂牌。
- 列7: 剧情弹幕串 - 每一个粉丝团挂牌都必须严格生成 3 到 6 条弹幕。专门吐槽、震惊、复盘、发疯、阴阳怪气或吃瓜当前剧情进展。必须有具体剧情内容，欢迎缺德、反串和带节奏。使用英文分号 ; 分隔。单条格式固定为 粉丝团挂牌：弹幕内容。每一个粉丝团挂牌严禁只输出1、2条。
- 列8: 推角弹幕串 - 每一个粉丝团挂牌都必须严格生成 3 到 6 条弹幕。站在该粉丝团立场给主推角色说话，但欢迎用拉踩对家、阴阳怪气、疯魔玩梗、造谣式吹捧、反串虐粉的方式护主。禁止平淡夸奖。使用英文分号 ; 分隔。单条格式固定为 粉丝团挂牌：弹幕内容。每一个粉丝团挂牌严禁只输出1、2条。
- 列9: 对线弹幕串 - 每一个粉丝团挂牌都必须严格生成 3 到 6 条弹幕。和其他粉丝团互喷、拉踩、阴阳怪气、抢功、反串、破防。可以嘴臭，像真实的粉圈骂战。使用英文分号 ; 分隔。单条格式固定为 粉丝团挂牌：弹幕内容。每一个粉丝团挂牌严禁只输出1、2条。
- 列10: 观看/互动数据 - 聚合文本，如“在线1.3w;弹幕24.6w;名场面剪辑8.7w;粉丝团对线1.2k;超话签到4.3w”。数据要有直播感但可信，不要每次爆表。可灵活加入超话相关数据。
- 列11: 时间文本 - 直接展示文本或开播时长。




# 弹幕写法
1. 单条弹幕建议 8 到 36 个中文字符，可以尖锐、缺德、好笑、粉圈化，但要像真实弹幕，不要像正式影评，剧情总结。
2. 粉丝团挂牌必须极短、有阵营感、好笑或恶毒，直接使用粉圈黑话。允许 AI 根据当前角色、剧情名场面、CP、黑称、梗自拟粉丝团挂牌，如“艾门永存”“主角事业粉”“雨夜伞党”“嘴硬研究所”“青梅保护协会”“反恋爱脑联盟”。
3. 剧情弹幕必须评论当前剧情：发生了什么、哪里反转、哪里尴尬、哪里好笑、哪里像大型事故现场、谁被下了面子、谁活该。要带节奏。
4. 推角弹幕必须带极强的粉籍立场，但绝不要“妹妹好温柔”“哥哥好帅”这种温吞夸奖。要用发疯、拉踩、反串、造谣式澄清、装路人被发现的方式护主。
5. 对线弹幕就照这个标准互骂、互扒皮、互甩黑料。要像失控的粉圈大战。
6. 禁止整格只输出1条弹幕冒充多条的偷懒行为。禁止整格“啊啊啊;哈哈哈;好甜;支持;演得真好”这种低密度废话；也禁止“本轮剧情表现出复杂关系推进”这种论文腔。必须带情绪和攻击性。
7. 不要用中文分号、顿号或换行冒充分隔。
8. 弹幕评论的是当前正文被包装成的实时直播剧：剧情怎么走、角色怎么演、对手戏有没有张力、谁被谁压戏、谁家粉丝破防。


# 数量与留存
1. 通常只维护 1 行直播剧主舞台信息，代表当前正在展示的实时片场。
2. 每轮优先更新已有行，不频繁新增。
3. 例如，如果有3个粉丝团挂牌。剧情弹幕串、推角弹幕串、对线弹幕串每一个弹幕串都要有9-18条。因为3*3=9。3*6=18。所以不要每一个弹幕串的每一个粉丝团挂牌只输出1、2条。

## sourceData.initNode

初始化时插入 1 条实时直播剧主舞台信息。默认把当前正文开场或最近剧情包装成片场飙戏现场。剧情弹幕串、推角弹幕串、对线弹幕串中的每一个粉丝牌必须生成3-6条弹幕。

## sourceData.deleteNode

默认不删除唯一主舞台。

## sourceData.updateNode

每轮正文推进后，优先更新当前这 1 行：更新时必须围绕最新正文内容和角色对手戏。

## sourceData.insertNode

通常不需要频繁新增。主表应保持 1 行为主，避免直播间列表膨胀。

## sourceData.ddl

```sql
CREATE TABLE livestream_rooms ( -- 直播表
  row_id INTEGER PRIMARY KEY, -- 行号
  room_name TEXT NOT NULL UNIQUE, -- 直播间名
  cast_lineup TEXT NOT NULL, -- 领衔阵容
  cast_tag TEXT NOT NULL, -- 阵容标签
  stream_title TEXT NOT NULL, -- 直播标题
  stage_summary TEXT NOT NULL, -- 剧情舞台概述
  chemistry_focus TEXT NOT NULL, -- 对手戏看点
  fandom_badge TEXT, -- 粉丝团挂牌
  plot_barrage TEXT, -- 剧情弹幕串
  stan_barrage TEXT, -- 推角弹幕串
  clash_barrage TEXT, -- 对线弹幕串
  interaction_text TEXT NOT NULL CHECK(TRIM(interaction_text) <> ''), -- 观看/互动数据
  time_text TEXT NOT NULL -- 时间文本
);
```
