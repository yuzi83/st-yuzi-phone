# styles/legacy 目录说明

该目录用于承接未来从 `styles/` 与 `styles/phone-base/` 中迁出的历史归档样式。

## 当前状态
- 当前 legacy 样式仍保留在原路径，目的是保持历史文档锚点、脚本护栏与人工检索路径稳定。
- 本目录在 Phase B 仅用于建立目标归档约定，不承载实际运行时样式加载。

## 迁移原则
1. 不恢复 legacy CSS 的默认加载。
2. 先建立目录说明与桥接约束，再移动 legacy 文件。
3. 物理迁移时，原路径应保留极薄桥接文件或迁移说明。
4. 至少完成一轮护栏与文档回归后，才评估删除原路径桥接层。

## 当前规划中的子目录
- `phone-base/`：承接原 `styles/phone-base/` 下的 legacy 样式归档。
